# Bugfixes Landing Page Messaging

## Core message hierarchy

1. Bugfixes is a signal-driven bug operations product, not a generic logging tool.
2. It turns stacktraces, errors, and policy-defined warning signals into normalized incidents and actionable engineering tickets.
3. Stacktraces are important because they are often the fastest route to what broke, but they are not the only trigger for workflow or alerts.
4. AI agents enrich each ticket with investigation context and fix guidance so engineers spend less time debugging from scratch.
5. It escalates repeat failures and alerts on org-defined severity thresholds so regressions do not disappear into event noise.
6. It fits into the team's existing backlog tools and remediation workflow.

## Plain-language product story

Bugfixes starts from the failure signal. Stacktraces are often the fastest route to understanding what broke, but Bugfixes also creates tickets from errors and alerts on warning or other severity levels based on the policy a team defines. Instead of asking engineers to search across raw logs first, Bugfixes groups duplicate incidents, keeps the context that matters, opens or updates the ticket, enriches it with AI-agent analysis and fix tips, and alerts the team when the same issue comes back.

## Primary calls to action

- Primary CTA: `Start Setup` -> `https://docs.bugfix.es`
- Secondary CTA: `Open Dashboard` -> `https://dashboard.bugfix.es`

The landing page should guide new visitors toward setup first. The dashboard is for users who already have the pipeline in place.
